SELF JOIN, VIEWS & INDEXES

SELF JOIN

A SELF JOIN is a join of a table with itself. It is useful when you want to compare rows within the same table.

Syntax:

SELECT a.column_name, b.column_name
FROM table_name a, table_name b
WHERE condition;

Example:

Consider an Employees table:

  • emp_id | emp_name | manager_id
  • 1 | Ramesh | NULL
  • 2 | Suresh | 1
  • 3 | Mahesh | 1
  • 4 | Naresh | 2
SELECT e1.emp_name AS Employee, e2.emp_name AS Manager
FROM Employees e1
LEFT JOIN Employees e2
ON e1.manager_id = e2.emp_id;

Result:

  • Ramesh | NULL
  • Suresh | Ramesh
  • Mahesh | Ramesh
  • Naresh | Suresh

VIEWS

A View is a virtual table based on the result of an SQL query. It does not store data itself but fetches data dynamically from the base tables.

Syntax:

CREATE VIEW view_name AS
SELECT column1, column2
FROM table_name
WHERE condition;

Example:

CREATE VIEW HighScorers AS
SELECT name, marks
FROM Students
WHERE marks > 80;

SELECT * FROM HighScorers;

INDEXES

An Index in SQL improves the speed of data retrieval operations. It works like the index of a book, allowing faster lookups.

Syntax:

CREATE INDEX index_name
ON table_name (column_name);

Example:

CREATE INDEX idx_student_name
ON Students(name);

This will make searching by student name faster.
