Views, Indexes & Subqueries

Views

A View is a virtual table based on the result of an SQL query. It stores no data itself but retrieves it dynamically from base tables.

Syntax:

CREATE VIEW view_name AS
SELECT column1, column2
FROM table_name
WHERE condition;

Example:

CREATE VIEW HighScorers AS
SELECT name, marks
FROM Students
WHERE marks > 80;

SELECT * FROM HighScorers;

Indexes

An Index improves query performance by making lookups faster, similar to an index in a book.

Syntax:

CREATE INDEX index_name
ON table_name (column_name);

Example:

CREATE INDEX idx_student_name
ON Students(name);

Subqueries

A Subquery is a query nested inside another query. It is used to perform operations that require results from another query.

Types of Subqueries:

  • Single-row Subquery – returns only one row.
  • Multi-row Subquery – returns multiple rows.
  • Nested Subquery – subquery inside another subquery.

Syntax:

SELECT column
FROM table
WHERE column operator (SELECT column FROM table WHERE condition);

Examples:

-- Find students with marks greater than the average
SELECT name, marks
FROM Students
WHERE marks > (SELECT AVG(marks) FROM Students);

-- Find employees earning more than employee 'Rahul'
SELECT emp_name, salary
FROM Employees
WHERE salary > (SELECT salary FROM Employees WHERE emp_name = 'Rahul');
