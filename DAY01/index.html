1. SQL Introduction

SQL (Structured Query Language) is used to communicate with relational databases. It helps in storing, retrieving, updating, and deleting data. SQL is standardized but implementations differ slightly across MySQL, PostgreSQL, Oracle, etc.

2. Why Use Databases?

  • Store large amounts of data systematically
  • Provide data security & integrity
  • Support multi-user access
  • Easy querying and analysis

3. Difference Between SQL & MongoDB

Feature SQL (Relational DB) MongoDB (NoSQL DB)
Data Model Tables (rows & columns) Documents (JSON-like)
Schema Fixed schema Schema-less
Relationships Supports JOINs Embedding/Nesting
Best For Structured data Unstructured / flexible data
Examples MySQL, PostgreSQL, Oracle MongoDB

4. Advantages of SQL

  • Simple & standardized language
  • Efficient for structured data
  • Powerful queries (JOINs, filtering, aggregation)
  • Security with roles & permissions
  • Scalable and widely used in industry

5. Basic Example

-- Create a database
CREATE DATABASE School;

-- Use the database
USE School;

-- Create a table
CREATE TABLE Students (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT,
    grade VARCHAR(5)
);

-- Insert data
INSERT INTO Students (id, name, age, grade)
VALUES (1, 'Ravi', 20, 'A');

-- Retrieve data
SELECT * FROM Students;
